# Universal Software Development Practices Knowledge Base

## 1. Software Development Philosophy
- Build maintainable and scalable systems.
- Focus on business value and user impact.
- Prefer iterative and incremental delivery.
- Design for long-term sustainability.
- Prioritize reliability and observability.

---

## 2. Requirement Analysis
Before implementation:
- Understand business goals.
- Clarify ambiguities.
- Identify functional requirements.
- Identify non-functional requirements.
- Define acceptance criteria.
- Identify risks and dependencies.
- Estimate complexity and effort.

---

## 3. System Design Practices
- Design modular architectures.
- Define clear service boundaries.
- Use separation of concerns.
- Design APIs/interfaces before implementation.
- Plan for scalability and failure handling.
- Use appropriate architectural patterns.
- Document system flows and dependencies.

---

## 4. Development Lifecycle
Typical lifecycle:
1. Requirement Gathering
2. Analysis
3. Architecture Design
4. Task Breakdown
5. Development
6. Testing
7. Security Review
8. Performance Validation
9. Deployment
10. Monitoring & Maintenance

---

## 5. Agile & Iterative Development
- Deliver in small increments.
- Continuously gather feedback.
- Maintain backlog prioritization.
- Track technical debt.
- Conduct retrospectives.
- Improve processes continuously.

---

## 6. Task Planning Standards
Tasks should:
- Be atomic and measurable.
- Have clear acceptance criteria.
- Include dependency tracking.
- Include testing requirements.
- Include rollback considerations.

---

## 7. Documentation Practices
Maintain documentation for:
- Architecture
- APIs
- Deployment
- Environment setup
- Security considerations
- Runbooks
- Troubleshooting guides

Documentation must remain updated.

---

## 8. Security Engineering Practices
- Follow secure-by-design principles.
- Apply least privilege access.
- Perform threat modeling.
- Conduct vulnerability scanning.
- Encrypt sensitive data.
- Use secure authentication mechanisms.
- Implement audit logging.
- Rotate credentials regularly.

---

## 9. Testing Strategy
Testing should include:
- Unit Testing
- Integration Testing
- Functional Testing
- Regression Testing
- Performance Testing
- Security Testing
- User Acceptance Testing

Testing must be automated whenever feasible.

---

## 10. CI/CD Practices
- Automate builds and testing.
- Use deployment pipelines.
- Enforce quality gates.
- Perform automated security scans.
- Use staging environments.
- Support rollback strategies.
- Validate deployments post-release.

---

## 11. Code Quality Management
- Enforce linting and formatting.
- Maintain code review process.
- Use static analysis tools.
- Monitor technical debt.
- Refactor regularly.
- Track code coverage.

---

## 12. Observability & Monitoring
Systems should provide:
- Centralized logging
- Metrics collection
- Health monitoring
- Alerting
- Distributed tracing
- Incident visibility

---

## 13. Reliability Engineering
- Design for fault tolerance.
- Avoid single points of failure.
- Use retries and circuit breakers.
- Implement graceful degradation.
- Prepare disaster recovery strategies.
- Define SLAs/SLOs where applicable.

---

## 14. Performance Engineering
- Measure performance continuously.
- Benchmark critical paths.
- Monitor latency and throughput.
- Optimize bottlenecks.
- Plan for peak traffic scenarios.

---

## 15. Deployment Practices
- Use environment isolation.
- Maintain infrastructure consistency.
- Automate deployments.
- Use infrastructure as code where possible.
- Validate environment configuration.
- Minimize deployment downtime.

---

## 16. Collaboration Practices
Teams should:
- Communicate clearly.
- Share ownership.
- Maintain transparency.
- Document decisions.
- Conduct knowledge sharing sessions.
- Encourage constructive reviews.

---

## 17. Incident Management
For production issues:
1. Detect quickly
2. Triage severity
3. Mitigate impact
4. Identify root cause
5. Implement fixes
6. Perform postmortem analysis
7. Prevent recurrence

---

## 18. AI Agent Task Execution Standards
When an AI agent receives a task:
1. Analyze the objective completely.
2. Understand project context.
3. Break work into subtasks.
4. Identify dependencies.
5. Validate implementation strategy.
6. Generate production-grade output.
7. Ensure security compliance.
8. Ensure testing coverage.
9. Verify compatibility with existing architecture.
10. Produce clear execution summaries.

---

## 19. Production Readiness Checklist
Before deployment:
- Security reviewed
- Tests passing
- Performance validated
- Logging enabled
- Monitoring configured
- Rollback strategy prepared
- Documentation updated
- Dependencies verified
- Secrets managed securely

---

## 20. Anti-Patterns to Avoid
- Tight coupling
- Poor error handling
- Lack of observability
- Manual deployments
- Unclear ownership
- Missing documentation
- Ignoring technical debt
- Overengineering
- Under-testing
- Security negligence

---

## 21. Decision-Making Guidelines
Choose solutions based on:
- Simplicity
- Maintainability
- Scalability
- Security
- Reliability
- Cost efficiency
- Developer productivity
- Long-term sustainability

---

## 22. Final Execution Validation
Before marking work complete:
- Are requirements satisfied?
- Is implementation scalable?
- Is security validated?
- Are tests included?
- Is documentation updated?
- Is deployment safe?
- Is monitoring enabled?
- Is rollback possible?
